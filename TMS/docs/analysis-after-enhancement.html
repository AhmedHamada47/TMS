3 — Enhancement Phases with Impact

Phase 6

Testing Infrastructure

Score Impact: +6
  • xUnit test project with EF Core in-memory provider
  • Integration tests for TasksController (CRUD, scoping, auth policies)
  • Unit tests for AccountController (login rate-limiting flow)
  • E2E smoke tests with Playwright
Impact: Testing goes from 1/10 to 9/10. Enables safe refactoring for all subsequent phases.
Phase 7

Service Layer Refactor

Score Impact: +2.5
  • ITaskService / ICategoryService / IReportService interfaces
  • ICurrentUserService abstraction for UserId / OrgId / Role
  • Authorization filter replacing inline `isAssigned || IsManagerOrAbove`
  • Duplicate code consolidated, TasksController reduced ~50%
Impact: Architecture 9 → 9.5, Code Quality 8 → 9.5. Future features become trivial to add.
Phase 8

Email Notifications

Score Impact: +0.5
  • MailKit SMTP integration
  • IHostedService background queue for email dispatch
  • Triggers: assignment, comment, due date approaching (24h)
  • Notification preference page (unsubscribe per type)
Impact: Feature completeness 8.5 → 9. Major user-facing value.
Phase 9

File Attachments

Score Impact: +0.5
  • Attachment entity linked to TaskItem and TaskComment
  • Local file storage with type/size validation
  • Upload UI on task Create / Edit / Details
Impact: Covers a common workflow gap. Enables rich task collaboration.
Phase 10

Real-Time Features (SignalR)

Score Impact: +1
  • SignalR hub for live notification push
  • Replace 30s polling with WebSocket
  • Optional: live Kanban board collaboration
Impact: UX 9 → 9.2. Notifications become instant. Sets stage for multi-user Kanban.
Phase 11

Production Readiness

Score Impact: +2
  • Migrate to PostgreSQL
  • Serilog structured logging
  • Health checks endpoint
  • Dockerfile + docker-compose
  • GitHub Actions CI/CD
  • Security headers (HSTS, CSP)
Impact: Performance 8 → 9, Security 8 → 9. Production-ready deployment pipeline.
Phase 12

Advanced Features

Score Impact: +1
  • Time tracking: start/stop timer, daily log
  • Full-text search (FTS5 / tsvector)
  • Recurring tasks
  • Task dependencies / blocking links
  • Calendar / Gantt view
  • Export to PDF / CSV / Excel
  • Role/permission management UI
Impact: Feature completeness 9.5 → 10. Differentiator features for enterprise adoption.

8 — Recommended Order

  1. Phase 6 (Tests) — Must come first. No refactoring without a safety net.
  2. Phase 7 (Service Layer) — Unlocks all subsequent phases by cleaning the architecture.
  3. Phase 8 (Email) — Highest user-facing value after the foundation is solid.
  4. Phase 10 (Real-Time) — Replace polling with SignalR (depends on Phase 5 notifications).
  5. Phase 9 (Attachments) — Straightforward; can be done in parallel with Phase 10.
  6. Phase 11 (Production) — Prerequisite for any live deployment.
  7. Phase 12 (Advanced Features) — Polish and differentiate after core is stable.
